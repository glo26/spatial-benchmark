# Contributing to SpatialEval

We welcome contributions from the community! Thank you for your interest in making SpatialEval better.

## How to Contribute

There are several ways you can contribute:

- **Reporting Bugs**: If you find a bug in the dataset or the code, please open an issue with a detailed description and steps to reproduce it.
- **Submitting Model Results**: To add your model to the leaderboard, please open a pull request with your evaluation results. Follow the format specified in the `submissions/` directory.
- **Proposing New Tasks**: If you have an idea for a new spatial reasoning task, please open an issue to discuss it with the community.
- **Improving Documentation**: If you find any part of the documentation unclear or incomplete, please submit a pull request with your improvements.

## Pull Request Process

1. Fork the repository and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`pytest`).
4. Make sure your code lints (`black .`).
5. Issue that pull request!

## Code of Conduct

We expect all contributors to adhere to our [Code of Conduct](./CODE_OF_CONDUCT.md). Please be respectful and constructive in all your interactions.
